# 依恋类型测试

基于ECR-R量表框架的依恋类型测试，MBTI同款交互体验。

🔗 **在线测试：** https://tsunegkt.github.io/attachment-style-test/

---

## 为什么做这个

依恋类型影响你在亲密关系中的每一次反应——如何表达爱、回应冲突、面对分离。了解自己的依恋模式，是改善关系的第一步。

但现有的依恋测试大多是四选一的简单问卷，选项颗粒度粗，难以捕捉真实感受的微妙差异。本项目从交互方式、题目设计、结果呈现三个层面重新设计了依恋类型测试。

## 特点

**答题交互**
- 采用MBTI（16personalities）同款7级渐变圆圈选项，从「同意」到「不认同」连续过渡
- 相比四选一，能更精确地捕捉用户态度的细微差异

**题目设计**
- 36道题，基于心理学ECR-R（亲密关系体验量表修订版）理论框架
- 题目以行为场景替代情绪自评（如「伴侣超过一小时没回消息时，你会忍不住看手机好几次」而非「我害怕被抛弃」），降低作答时的防御性
- 正向题与反向题混合，防止惯性作答
- 测量焦虑与回避两个独立维度，每个维度下设三个子维度（共六个）

**结果呈现**
- 四种依恋类型（安全型、焦虑型、回避型、恐惧回避型）各配有专属低多边形角色形象，男女各一版
- 焦虑×回避四象限定位图，标出你的精确位置
- 六子维度雷达图（害怕被抛弃、渴望确认、过度敏感、抗拒亲密、抗拒依赖、情感封闭）
- 百分制分数 + 程度标签（偏低/中等偏低/中等偏高/偏高）
- 每种类型提供描述、优势分析和成长方向

**AI 深度分析**
- 支持导出完整作答记录为 Markdown（复制到剪贴板或下载文件）
- 导出内容包含测试结果、六子维度分数，以及按子维度分组的逐题作答记录（含反向题标注）
- 内置提示词，用户可将记录粘贴给任意 AI（DeepSeek、豆包、Kimi、ChatGPT 等），获得基于每一道题回答的个性化分析

## 理论基础

本测试参考 Fraley 等人开发的 ECR-R（Experiences in Close Relationships-Revised）量表的理论框架，测量两个独立维度：

- **依恋焦虑**：对被抛弃的恐惧程度、对关系确认的需求程度、对伴侣行为的敏感程度
- **依恋回避**：对亲密接触的舒适程度、对依赖他人的接受程度、情感开放的程度

两个维度的高低组合决定四种依恋类型：

| | 低回避 | 高回避 |
|---|---|---|
| **低焦虑** | 安全型 | 回避型 |
| **高焦虑** | 焦虑型 | 恐惧回避型 |

> ⚠️ 本测试仅供自我探索参考，不构成心理诊断。如需专业帮助，请咨询心理咨询师。

## 技术实现

- 纯前端，单个 HTML 文件 + 图片资源，零后端依赖
- 原生 JavaScript，无框架
- Canvas API 生成分享图（无外部库依赖）
- 客户端生成 Markdown 作答记录，剪贴板复制与文件下载均在前端完成，AI 分析借助用户自有的大模型工具，项目本身无 API 成本
- 字体：正文使用系统黑体，标题异步加载思源宋体（CDN不可用时自动回退系统宋体）
- 部署于 GitHub Pages
